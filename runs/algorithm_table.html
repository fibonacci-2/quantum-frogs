What it stores
Q-table — a dictionary mapping every observed state (raw bytes of the obs array) to a vector of 5 action values.
Grows with unique states visited. Works only because Stage 1–2 has a small, discrete state space.
Two networks — online Q-net + target Q-net (both MLP 192→256→256→5).
Replay buffer — ring buffer of 100 k transitions (s, a, r, s′, done).
Two independent DQN agents, each with its own online net, target net, and replay buffer (100 k each).
No shared memory between agents.
Two actor nets (one per frog) + one shared critic.
Rollout buffer — on-policy trajectories from 32 envs × 128 steps = 4 096 transitions, discarded after each update.
Action selection
ε-greedy with multiplicative decay (ε × 0.9995 each episode).
a = argmax Q(s,·) w.p. 1−ε a = random w.p. ε
ε-greedy with linear schedule (1.0 → 0.05 over 30 % of training).
a = argmax Q_net(s) w.p. 1−ε a = random w.p. ε
Each agent independently runs ε-greedy on its own Q-net. They never consult each other.
aA = argmax QA(s) ← frog A's net aB = argmax QB(s) ← frog B's net
Stochastic sampling from the actor's categorical distribution. No ε needed — entropy bonus keeps exploration alive.
aA ~ Categorical(πA(s)) aB ~ Categorical(πB(s))
Update rule
Bellman update on every step. No mini-batch, no replay — one transition updates one table entry directly.
Q(s,a) ← Q(s,a) + α · [r + γ·max Q(s′,·) − Q(s,a)]
α = 0.1, γ = 0.99
Sample a random mini-batch from replay. Compute TD target using frozen target net. MSE loss, Adam optimiser.
target = r + γ · max Q_tgt(s′,·) · (1−done) loss = MSE(Q_net(s,a), target) Q_tgt ← Q_net every 1 000 steps
batch=128, lr=1e-3, train every 4 steps
Same as DQN but run twice — once per agent — each on its own replay buffer and its own reward signal.
lossA = MSE(QA(s, aA), rA + γ·max QA_tgt(s′)) lossB = MSE(QB(s, aB), rB + γ·max QB_tgt(s′))
rA and rB are individual rewards, not shared
Collect rollout → GAE advantages → 4 PPO epochs with clipped surrogate objective. One optimiser for all parameters.
δt = r_team + γ·V(s′) − V(s) A_t = δt + (γλ)δt+1 + (γλ)²δt+2 + … L_actor = −min(ratio·A, clip(ratio,1±ε)·A) L_critic = MSE(V(s), returns) L_entropy = H[πA] + H[πB] loss = L_actor + 0.5·L_critic − 0.01·L_entropy
clip ε=0.2, λ=0.95, 4 epochs, batch=512
Reward signal
Individual frog reward: +100 reach top, −100 hit, −1 per step, +1 per row gained upward.
Same individual reward as Q-Learning. Single agent, so there is no ambiguity.
Each agent trains on its own reward only (rA or rB). Neither agent sees or optimises for the other's outcome.
This is the "independent" in IDQN — the source of instability.
Team reward: rA + rB, shared equally. Both actors receive the same scalar and learn to maximise the joint outcome.
This is what drives cooperative behaviour — not explicit messaging.
Key mechanism
Lookup table + Bellman. Every (state, action) pair gets its own learned value. Exact — no generalisation. The table is the policy.
Experience replay + target network. Replay breaks temporal correlation; the target net provides a stable regression target that doesn't chase itself.
Non-stationarity. From each agent's perspective the environment changes as the other agent learns — making convergence unreliable. The +34 pp gap vs MAPPO is the cost of this instability.
Centralised Training, Decentralised Execution (CTDE). The critic sees global state during training and teaches actors how joint actions affect joint returns. At test time, each actor acts alone using only its own obs.
Generalises to unseen states?
No. New state → no table entry → falls back to random. Must visit every state to learn about it.
Yes. The network interpolates across similar observations. Trained on 4 cars, still works on 1–6 cars at eval.
Yes, same as DQN — but generalisation is disrupted if the other agent's behaviour is still shifting.
Yes. Trained on 4 cars, evaluated on 1–6. Actors generalise cleanly because the critic's value signal is stable.
Convergence
Guaranteed under standard conditions (visiting all states, α schedule). Converges to optimal Q* for small state spaces.
No guarantee (function approximation + off-policy = deadly triad). In practice stable enough with replay + target net.
No guarantee. Non-stationarity breaks the theoretical requirements. Observed: 7× spread across 4 seeds (10 % to 79 %).
No guarantee for global optimum, but PPO's clipping keeps updates small. Observed: all 4 seeds converged to identical policies — far more stable than IDQN.
